# Loading a Track

If you already have an Audio or Video element object then you can load an initial track or a new track into that object by changing the value of the src attribute.

If you are dealing with an Audio or Video element in the HTML and the <source> child elements have been changed then you could also call the load() method which will read all the <source> elements, select the best one and restart the audio or video loading or playing.

In our project, since we are dealing with a single Audio element that is created in the JavaScript, we will just be calling the pause() method to stop the current track playing and then changing the value of the src attribute to load the new track.

When the page loads, your Audio object needs to load the first track from the media array.

# Handling Audio Track Errors

Any time you load an Audio track into an Audio or Video element then you should ALWAYS have an error handler. While this can be accomplished by adding try{ }catch(){ } blocks into your code, that can be hard for other developers to read. The best practice is to add an actual error event listener. The error event is fired when the resource could not be loaded due to an error.

myAudioElementObject.addEventListener('error', errorHandler);

//...

function errorHandler(ev) {
  //do what you want when the audio/video track cannot load
  // ev.type = 'error'
}

# Playing and Pausing

The 3 controls in the fixed footer are previous track, play | pause, and next track. In this second phase we need to get the play and pause feature working.

In the footer only one of the icons for play and pause should be visible at a time. If the audio is currently playing then the pause icon should be visible. If the audio is not playing then the play icon should be visible.

The button that wraps around the icon can change which one is displayed by changing the text inside the <i> element. To show the play icon the text would be play_arrow.

<button id="btnPlay" title="play | pause"><i class="material-symbols-rounded">play_arrow</i></button>
1

In the JavaScript the click listener for all three buttons should be on the <div class="controls"> element. Based on the ev.target we can determine which of the three buttons were clicked. Remember that the user could have clicked an <i> or a <button>.

An HTML Audio element inherits all its properties and events and methods from the HTML Media Element. It includes a paused property which you can use to know if an audio track is currently playing or not.

To play or pause an Audio track we only have to call the play() or pause() method.

# Large Image

Whenever a new Audio track is loaded into APP.audio, we also need to load the large image into the .album_art__full img element. In the final phase this image will be animated. For now, we just want to display the larger version of the thumbnail image any time the track changes. For this reason it should be part of the code that runs to load each track by setting APP.audio.src value.

# Displaying Current Time

HTML Media Elements have a timeupdate event and a currentTime property.

Add a listener for the timeupdate event to update the current time display in the footer. This event will be triggered multiple times per second as the current time in the track changes.

The currentTime property will give you the exact number of seconds as a decimal value. You can remove the decimal part of the number to get an integer and then convert this number to the 00:00 style display for the footer.

# Displaying Total Times

HTML Media Elements also have a canplay event which will tell you if enough of the media file has been loaded that it can start playing from the current position. When you first load a track it's current position will be zero. An alternative event that can be used for this is the loadedmetadata event. It fires when all the meta data for the track has been downloaded and read. There is also a durationchange event that fires when the duration of the track is available.

When it is able to play (or the durationchange event fires), then you will be able to determine the total length of the current track by using the duration property.

Once you have the duration value, then you can convert it to the 00:00 format, just like the current time, and display it in the footer.
