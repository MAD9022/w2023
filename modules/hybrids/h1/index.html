Media Player

MAD9022 Hybrids Phase 1

# Hybrid Overview

This semester the Hybrid portion of MAD9022 will be the construction of a fully functional Media Player web app. There will be four submissions. Each submission will build on the previous one.

You will need to create a single Github repo and then use a different branch for each phase in the process. Github Pages will be used to host the media player. Each phase, after the first, will require a merge of the current phase branch into the main branch.

# Designing a Media Player

The first phase will focus on the design, the gathering of assets, preparation of media files, plus the HTML and CSS.

The media player will need to have a responsive layout. The primary parts of the interface are a fixed header with a title, a fixed footer with the media controls, a fixed progress bar that sits on top of the footer, the playlist, and a media area that will display album artwork and animations when playing.

The layout can be constructed from this basic structure.

# Phase One Requirements

# General Requirements

This first phase should be in a Github Repo called media-player and use the branch main (NOT master).

All the CSS files should be in a css folder. All the Javascript files should be in a js folder. There should be a media folder to hold the mp3 files. All images should be inside an img folder.

All files and folders should be in lowercase and have no spaces.

There should be an external web font loaded from a CDN.

No CSS libraries are to be used.

You must write your own CSS.

You should have a colour scheme defined with a series of CSS variables.

The index.html file should have a Content-Security-Policy meta tag that allows files from self and from the font CDN.

The icons used on the controls should come from one of the Google Material Design Icon Fonts on https://fonts.google.com/icons (opens new window).

Each of the controls on the page should have an HTML title attribute indicating what the icon is.

Your project should have a .gitignore file.

Your project should have a custom favicon image in the root of the project and a <link> tag that loads it.

# mp3 Files

You will need at least 7 mp3 files of songs that you like. The mp3 files should be at least 1 minute long. You can use any online tool you want to extract the mp3 files from YouTube if you need.

Keep the filename lowercase with no spaces. The files should all be inside the media folder.

# Images

For each of the songs that you have in the media player you will need two images. One will be roughly 100px squared. The other will be between 600px and 800px, squared.

The images should be in the img folder and have lowercase names with no spaces.

# Data File

You need to create a JavaScript data file with all the information about your songs.

Eg:

const MEDIA = [
  {
    artist: 'The Offspring',
    title: 'Come out and play',
    track: 'come-out-and-play.mp3',
    thumbnail: 'offspring-thumb.jpg',
    large: 'offspring-full.png',
  },
];
export default MEDIA;

# HTML and CSS

For the first phase we need to do the design work. We need to make sure that the layout is responsive and works at all screen dimensions. No new topics need to be taught for you to complete this phase. The course material for MAD9013 and MAD9014 covered everything you need to complete this assignment.

For this first phase you will hard-code the HTML and data from the Media Data File into the HTML playlist. In the next phase this hard-coded data will be generated by the JavaScript.

In your CSS for the default data, a single track in the playlist should be set and styled as the current track. We can do this with a single class name on the <li>.

No functionality is needed for this phase, just the responsive design and all the assets.

You must deploy your private repo with Github Pages. This means that you must have signed up for the Github Education Pack (opens new window). You should have already done this in first semester.
