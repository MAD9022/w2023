# Fully Functional Media Player

In the third phase of this project we will calculate the duration of each of the songs in order to update the playlist display. We will add the automated loading of the next track when one track ends. We will add the ability for the user to click on any track in the playlist and have that stop the currently playing track and load the clicked one. The users will need to be able to click the next and previous track buttons and move to the next track.

For this third phase you need to create a new branch called tracks based on the updated main branch.

Make sure that you have merged the play branch back into main before creating the tracks branch.

# Display Track Durations

In Phase 2 we displayed the track duration for the current track in the footer. We had to wait for the canplay or loadedmetadata or durationchange event and then convert the number of seconds to the 00:00 style display.

To update the playlist we need to repeat the same thing for each item in the MEDIA array. Here is an example showing how to load a series of urls into a series of Audio elements, and add an event listener function to each one.

function one() {
  arrayOfAudioTracks.forEach((track) => {
    //create a temporary audio element to open the audio file
    let tempAudio = new Audio(`./path/${track.url}`);
    //listen for the event
    tempAudio.addEventListener('durationchange', (ev) => {
      //`tempAudio` and `track` are both accessible from inside this function
      //update the array item with the duration value
      let duration = ev.target.duration;
      track['duration'] = duration;
      //update the display by finding the playlist item with the matching img src
      //or track title or track id...
      let thumbnails = document.querySelectorAll('.track__item img');
      thumbnails.forEach((thumb, index) => {
        if (thumb.src.includes(track.thumbnail)) {
          //convert the duration in seconds to a 00:00 string
          let timeString = convertSecondsToTime(duration);
          //update the playlist display for the matching item
          thumb.closest('.track__item').querySelector('time').value = timeString;
        }
      });
    });
  });
}

If you put a data- attribute inside each <li> that holds the name of the track media file, then we can use that as part of your CSS query. In the code sample above, track.url is the name of the file that you can use as part of the selector.

let timeElement = document.querySelector(`li[data-source="${track.url}"] time`);

For the media player moving from one track to the next means:

  • stop the current track playing
  • changing the value of the currentTrack property by incrementing or decrementing it.
  • if the value of currentTrack is less than zero or greater than or equal to the length of the array, then loop back to the index at the opposite end of the array.
  • load the source from the new currentTrack entry in the media array into the Audio element.

Your script should have a next and a previous function that are responsible for incrementing and decrementing the current track value.

Here is an example of a next function.

const APP = {
  currentTrack: 0,
  audio: new Audio(),
  /* ... */
  next: () => {
    APP.audio.pause(); //stop the current track playing
    APP.currentTrack++; //increment the value
    if (APP.currentTrack >= MEDIA.length) {
      APP.currentTrack = 0;
    }
    //call the function to load the MEDIA[APP.currentTrack] src into APP.audio.src
    //then call your function to play the new track
  },
};

# Click Playlist to Select Track

As is common best practice, we only want to add one click listener to the playlist container. This is most likely a <ul> and each of the items in the list are <li> items. When the user clicks a playlist item, the click could be starting on the thumbnail image, the artist name, the track name, or the li itself.

So, when the user clicks the <ul>, we use the closest() method to find the actual <li>. Once you have the <li> we can read the value of any attribute or descendant element text.

A good approach here would be to add a data- attribute inside each <li> when you are dynamically building the playlist. Add the src for the track as the value for this attribute.

With element.getAttribute('data-source') you can grab the name of the mp3 to load. Next, loop through the MEDIA array with the find() or findIndex() method to get the array item for your new current track. Then stop the current track playing with APP.audio.pause(). Then update APP.currentTrack with your new index number, and finally, reuse your function to load MEDIA[currentTrack].src into APP.audio.src.

# Automated Play of Next Track

This functionality will work the same way as your next button. We need to call the next function but are not listening for a click event.

Instead, we need to add a listener for the ended event to the APP.audio object. The audio track will automatically fire the ended event when it gets to the end of the currently playing audio track.

We should already have all the needed functions in our code to actually increment the currentTrack integer and play the next track.
